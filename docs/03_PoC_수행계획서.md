> **수정일**: 2026-01-30 (동적 레이아웃 생성 방식 전환)

## 1. PoC 개요

### 1.1 목적

사용자가 **폼 입력**으로 상품 정보를 제공하면, AI가 **동적으로 레이아웃을 생성**하고 **텍스트를 코드로 오버레이**하여 **웹 상세페이지를 즉시 생성**하는 시스템의 기술적 타당성 검증

> **핵심 원칙**: 레이아웃 동적 생성 + 텍스트 정확도 확보

### 1.2 핵심 변화 (v4.0)

| 항목 | 이전 (v3.0) | 현재 (v4.0) |
|------|-------------|-------------|
| **핵심 가치** | AI 텍스트 생성 | **동적 레이아웃 생성** |
| **텍스트 처리** | AI가 생성 | **코드로 오버레이** |
| **AI 역할** | 카피라이팅 | **비주얼 레이아웃 생성** |
| **출력 형태** | 이미지 파일 | **웹 기반** (HTML/CSS) |

### 1.3 성공 기준

| 항목 | 기준 | 측정 방법 |
|------|------|-----------|
| **레이아웃 생성 데모** | 섹션 프롬프트 3종 이상으로 레이아웃 생성 | 실제 데이터로 시연 |
| **레이아웃 품질** | 수작업 대비 80% 이상 미적 완성도 | 현업 담당자 평가 |
| **텍스트 오버레이** | 좌표 기반 텍스트 배치 정확도 90% 이상 | 기능 테스트 |
| **전체 생성 시간** | 30초 이내 | 입력 완료 → 결과 표시 |
| **웹 출력** | 반응형 웹 페이지 정상 렌더링 | 출력 테스트 |

### 1.4 범위

#### ✅ 포함 (In Scope)

| 항목 | 상세 |
|------|------|
| **출력물** | 웹 기반 상세페이지 (텍스트 오버레이) |
| **입력 방식** | **폼 입력** (엑셀은 Nice-to-have) |
| **레이아웃** | 섹션 프롬프트 3종 이상 (single, grid_2, grid_3) |
| **AI 파이프라인** | Gemini (분석/좌표추출) + Nano Banana Pro (이미지) |
| **사용자 플로우** | **입력 → [AI 파이프라인] → 결과 확인 → 웹 출력** |
| **편집** | 텍스트 수정 (선택 사항) |
| **기술 스택** | Next.js + FastAPI + Supabase |

#### ❌ 제외 (Out of Scope)

- 이메일 자동 수집 (RPA)
- 기존 POP 유형 (샤넬 30×30, DID 등)
- 고급 편집 (드래그앤드롭)
- 사용자 인증/권한 관리
- 프로덕션 배포
- Claude 텍스트 생성 (텍스트는 사용자 입력 사용)

## 3. AI 파이프라인 상세

### 3.1 파이프라인 흐름

```
[1] 사용자 입력
    │  - 상품 정보 (이름, 가격)
    │  - 상품 이미지
    ▼
[2] Gemini: 입력 분석 → 섹션 프롬프트 선택
    │  - 상품 개수, 카테고리 분석
    │  - 적합한 레이아웃 섹션 선택
    ▼
[3] Nano Banana Pro: 레이아웃 이미지 생성
    │  - 섹션 프롬프트 + 상품 이미지
    │  - 텍스트 없는 순수 비주얼 생성
    ▼
[4] Gemini Vision: 텍스트 영역 좌표 추출
    │  - 생성된 이미지 분석
    │  - 텍스트 배치 가능 영역 좌표 반환
    ▼
[5] 코드: 텍스트 오버레이
    │  - 좌표 기반 HTML/CSS 텍스트 배치
    ▼
[6] 웹 상세페이지 출력
```

### 3.2 프롬프트 구성

| 프롬프트 유형 | 개수 | 용도 |
|--------------|------|------|
| **레이아웃 생성 프롬프트** | 3종 | single_feature, grid_2, grid_3 |
| **텍스트 위치 분석 프롬프트** | 1개 (범용) | 모든 이미지에 적용 |

### 4.2 최종 데모 시나리오

```
[데모 플로우 - 약 3분]

1️⃣ 입력 (30초)
   ├── 브랜드명: "헤라"
   ├── 상품 3개 입력 (이름, 가격)
   ├── 상품 이미지 3장 업로드
   └── [생성하기] 버튼 클릭

2️⃣ AI 파이프라인 실행 (30초)
   ├── 로딩 화면 표시
   │   ├── "입력 분석 중..." (Gemini)
   │   ├── "레이아웃 생성 중..." (Nano Banana Pro)
   │   └── "텍스트 영역 분석 중..." (Gemini Vision)
   └── 생성 완료 → 결과 화면 자동 이동

3️⃣ 결과 확인 (60초)
   ├── 생성된 레이아웃 이미지 표시
   ├── 텍스트 오버레이 확인 (브랜드명, 상품명, 가격)
   ├── (선택) 텍스트 클릭 → 수정 시연
   └── 레이아웃 품질 확인

4️⃣ 재생성 옵션 (30초)
   ├── [다시 생성] 버튼 클릭
   └── 다른 레이아웃으로 재생성 확인

5️⃣ 웹 출력 (30초)
   └── 반응형 웹 페이지로 표시 확인
```
