## 1. PoC 개요

### 1.1 목적

사용자가 **상품 정보 + 테마**를 입력하면, **템플릿 기반**으로 **AI가 배경과 카피를 생성**하여 **편집 가능한 상세페이지**를 즉시 생성하는 시스템의 기술적 타당성 검증

> **핵심 원칙**: 템플릿 기반 렌더링 + AI 콘텐츠 생성 + 편집 가능한 결과물

### 1.2 핵심 변화 (v5.0)

| 항목 | 이전 (v4.1) | 현재 (v5.0) |
|------|-------------|-------------|
| **핵심 접근** | 동적 레이아웃 생성 | **템플릿 기반 렌더링** |
| **입력 데이터** | 브랜드, 상품, 카테고리, 토글 옵션 | **상품명, 가격, 이미지, 테마 (4개)** |
| **AI 역할** | 레이아웃 생성 + 좌표 추출 + 카피 | **배경 생성 + 카피 생성** |
| **출력 형태** | 이미지 + 텍스트 오버레이 | **편집 가능한 템플릿 결과** |
| **좌표 추출** | Gemini Vision 사용 | **불필요 (템플릿에 슬롯 정의)** |

### 1.3 v5.0 간소화된 흐름

```
[입력]                    [AI 처리]                   [출력]
 4개 필드만          ┌─────────────────────┐      편집 가능한
 - 상품명      ───▶  │ 1. 템플릿 자동 선택   │ ───▶  결과물
 - 가격              │ 2. 배경 생성 (AI)    │      (Canva 스타일)
 - 이미지            │ 3. 카피 생성 (AI)    │
 - 테마              │ 4. 슬롯에 데이터 바인딩│
                     └─────────────────────┘
```

### 1.4 성공 기준

| 항목 | 기준 | 측정 방법 |
|------|------|-----------|
| **템플릿 렌더링** | 3종 템플릿 정상 렌더링 | 실제 데이터로 시연 |
| **테마 배경 품질** | 테마별 일관된 배경 생성 80% 이상 | 현업 담당자 평가 |
| **캠페인 카피 품질** | 테마와 어울리는 카피 생성 80% 이상 | 현업 담당자 평가 |
| **전체 생성 시간** | 20초 이내 | 입력 완료 → 결과 표시 |
| **편집 기능** | 슬롯별 텍스트/이미지 수정 가능 | 기능 테스트 |

### 1.5 범위

#### ✅ 포함 (In Scope)

| 항목 | 상세 |
|------|------|
| **출력물** | 편집 가능한 웹 기반 상세페이지 |
| **입력 방식** | **폼 입력 (4개 필드)** |
| **템플릿** | 3종 (single_product, grid_2, grid_3) |
| **테마** | 5종 (홀리데이, 봄/여름/가을/겨울) |
| **AI 처리** | 테마 배경 생성 (Nano Banana Pro) + 카피 생성 (Gemini) |
| **Happy Case** | **단일 상품 + 봄 세일 테마** |
| **기술 스택** | Next.js + FastAPI + Supabase |

#### ❌ 제외 (Out of Scope)

- Gemini Vision 좌표 추출 (템플릿으로 대체)
- 동적 레이아웃 생성 (템플릿 기반으로 변경)
- 토글 옵션 (배송/안내/이벤트)
- 이메일 자동 수집 (RPA)
- 사용자 인증/권한 관리
- 프로덕션 배포

## 4. AI 파이프라인 상세 (v5.0)

### 4.1 파이프라인 흐름

```
[1] 사용자 입력 (4개 필드)
    │  - 상품명
    │  - 가격
    │  - 상품 이미지 (PNG/JPG)
    │  - 테마 선택 (5종 중 1개)
    ▼
[2] 템플릿 자동 선택 (코드 로직)
    │  - 상품 1개 → single_product
    │  - 상품 2개 → grid_2
    │  - 상품 3개+ → grid_3
    │  ⚠️ AI 분석 불필요 (단순 규칙)
    ▼
    ┌────────────────┬────────────────┐
    ▼                                ▼
[3-A] Gemini            [3-B] Nano Banana Pro
 캠페인 카피 생성          테마 배경 생성
 • 테마 키워드 기반        • 테마 프롬프트 사용
 • 15자 이내              • 텍스트 없는 배경
    │                                │
    └────────────────┬────────────────┘
                     ▼
[4] 템플릿 렌더링
    │  - 선택된 템플릿 로드
    │  - 슬롯에 데이터 바인딩
    │    • background ← AI 생성 배경
    │    • campaign_copy ← AI 생성 카피
    │    • product_image ← 사용자 이미지
    │    • product_name ← 사용자 입력
    │    • price ← 사용자 입력
    ▼
[5] 편집 가능한 결과물 출력
    • 각 슬롯 클릭 → 편집 가능
    • 이미지/텍스트 교체 가능
    • 최종 이미지 내보내기
```

### 4.2 v4.1 대비 제거된 단계

| 제거된 단계 | 이유 |
|------------|------|
| Gemini 입력 분석 | 상품 개수로 템플릿 결정 (단순 규칙) |
| Gemini Vision 좌표 추출 | 템플릿에 슬롯 위치 미리 정의됨 |
| 복잡한 섹션 프롬프트 선택 | 템플릿이 레이아웃 담당 |
| 토글 옵션 처리 | 입력 간소화로 제거 |

### 4.3 AI 처리 시간 예상

| 단계 | 예상 시간 | 비고 |
|------|----------|------|
| 카피 생성 (Gemini) | ~3초 | 병렬 처리 |
| 배경 생성 (Nano Banana Pro) | ~15초 | 병렬 처리 |
| 템플릿 렌더링 | ~1초 | 코드 처리 |

| **슬롯 편집기 복잡도** | 중간 | 낮음 | 텍스트만 우선 구현, 이미지 교체는 후순위 |
| **Nano Banana Pro 연동 실패** | 높음 | 낮음 | 대체 이미지 생성 API 확보 |
| **일정 지연** | 중간 | 중간 | 슬롯 편집기 제외, 결과 표시만 집중 |

### 7.2 일정 지연 시 대응

**Phase 1 (필수)**: W4까지
- 입력 → AI 파이프라인 → **결과 표시** (슬롯 편집 없이)
- Happy Case 1개 완성

**Phase 2 (선택)**: W5
- 슬롯 편집기
- 이미지 내보내기
- 프롬프트 튜닝

최악의 경우 Phase 1만 완성하여 "템플릿 기반 AI 생성 데모"로 진행
