## 1. 전체 사용자 플로우 (v5.0 - 템플릿 기반)

> **핵심 원칙**: 섹션 템플릿 + AI 배경/카피 생성 + 편집 가능한 결과

```
┌───────────────────────────────────────────────────────────────────────────────────────────┐
│                               사용자 Journey (v5.0)                                        │
└───────────────────────────────────────────────────────────────────────────────────────────┘

     [1단계]                    [2단계]                   [3단계]            [4단계]
    단순 입력                AI 파이프라인              템플릿 렌더링        편집/출력
    ────────                ──────────────             ────────────        ────────

  ┌─────────────┐         ┌─────────────────┐       ┌─────────────┐   ┌─────────────┐
  │ 폼 입력     │         │ AI 자동 처리     │       │ 템플릿에     │   │  편집 가능   │
  │ ───────    │         │                 │       │ 데이터 바인딩│   │  결과물      │
  │ • 상품명   │         │ 1. 템플릿 선택   │       │ ───────────│   │             │
  │ • 가격     │ ─────▶  │    (상품 개수)   │ ───▶  │ • 배경     │ ─▶│ [Canva처럼] │
  │ • 이미지   │         │ 2. Gemini       │       │   (AI생성) │   │  수정 가능   │
  │ • 테마선택 │         │    카피 생성     │       │ • 상품정보 │   │             │
  │            │         │ 3. Nano Banana  │       │   (입력값) │   │             │
  │            │         │    배경 생성     │       │ • 카피     │   │             │
  └─────────────┘         └─────────────────┘       │   (AI생성) │   └─────────────┘
                                                    └─────────────┘
```

### 기존 vs 신규 플로우 비교

| 항목 | 기존 (v4.1) | 신규 (v5.0) |
|------|-------------|-------------|
| **핵심 가치** | 동적 레이아웃 생성 | **템플릿 기반 렌더링** |
| **입력 방식** | 확장 입력 (토글 옵션) | **단순 입력 (4개 필드)** |
| **레이아웃** | AI가 매번 이미지 생성 | **사전 정의 템플릿 사용** |
| **텍스트 위치** | Gemini Vision 좌표 추출 | **템플릿에 슬롯 정의** |
| **AI 역할** | 레이아웃 + 카피 생성 | **배경 + 카피 생성** |
| **결과 편집** | 텍스트만 수정 | **이미지/텍스트 모두 편집** |

## 2. 단계별 상세 설계

### 2.1 STEP 1: 입력 (단순 폼)

#### 입력 필드 (4개만)

| 필드 | 타입 | 필수 | 설명 | 예시 |
|------|------|------|------|------|
| `product_name` | text | ✅ | 상품명 | "클래식 볼캡 스카이블루" |
| `price` | text | ✅ | 가격 | "45,000원" |
| `image` | file | ✅ | 상품 이미지 (PNG/JPG) | hat.png |
| `theme` | select | ✅ | 테마 선택 | "봄 세일" |

#### 테마 옵션 (5개)

| 테마 | 설명 | 배경 키워드 |
|------|------|------------|
| 홀리데이 | 크리스마스/연말 | 크리스마스 트리, 따뜻한 조명, 축제 분위기 |
| 봄 세일 | 봄 시즌 | 벚꽃, 파스텔톤, 산뜻한 느낌 |
| 여름 특가 | 여름 시즌 | 시원한 블루, 바다, 청량감 |
| 가을 신상 | 가을 시즌 | 단풍, 따뜻한 브라운톤 |
| 겨울 프로모션 | 겨울 시즌 | 눈, 화이트, 고급스러운 느낌 |

## 3. 섹션 템플릿 시스템

### 3.1 템플릿 구조

```
┌─────────────────────────────────────────────────────────────────────────┐
│  섹션 템플릿 = HTML/CSS 파일 + 슬롯 정의 (JSON)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   📁 templates/                                                         │
│   ├── single_product/                                                   │
│   │   ├── template.html      ← 레이아웃 HTML/CSS                       │
│   │   └── slots.json         ← 슬롯 정의 (위치, 바인딩)                │
│   ├── grid_2/                                                           │
│   │   ├── template.html                                                 │
│   │   └── slots.json                                                    │
│   └── grid_3/                                                           │
│       ├── template.html                                                 │
│       └── slots.json                                                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 템플릿 예시 (single_product)

#### HTML 템플릿

```html
<!-- templates/single_product/template.html -->
<div class="template-container" style="position: relative; width: 1080px; height: 1350px;">

  <!-- 배경 레이어: AI 생성 -->
  <img
    class="slot background-slot"
    data-slot="background"
    src="{{background}}"
    style="position: absolute; inset: 0; width: 100%; height: 100%; object-fit: cover;"
  />

  <!-- 캠페인 카피: AI 생성 -->
  <div
    class="slot text-slot"
    data-slot="campaign_copy"
    style="position: absolute; top: 10%; left: 50%; transform: translateX(-50%);
           font-size: 48px; font-weight: bold; color: #1a1a1a; text-align: center;"
  >
    {{campaign_copy}}
  </div>

  <!-- 상품 이미지: 사용자 업로드 -->
  <img
    class="slot image-slot"
    data-slot="product_image"
    src="{{product_image}}"
    style="position: absolute; top: 30%; left: 50%; transform: translateX(-50%);
           width: 400px; height: 400px; object-fit: contain;"

  <div

  </div>

</div>
```

### 3.3 템플릿 선택 로직

```python
def select_template(product_count: int) -> str:
    """상품 개수 기반 템플릿 자동 선택"""
    if product_count == 1:
        return "single_product"
    elif product_count == 2:
        return "grid_2"
    else:
        return "grid_3"
```

## 7. Happy Case (상품 1개)

### Input

| 필드 | 값 |
|------|-----|
| 상품명 | 클래식 볼캡 스카이블루 |
| 가격 | 45,000원 |
| 이미지 | hat.png |
| 테마 | 봄 세일 |

### Flow

```
[1] 입력
    │  상품명: "클래식 볼캡 스카이블루"
    │  가격: "45,000원"
    │  이미지: hat.png
    │  테마: "봄 세일"
    ▼
[2] 템플릿 선택
    │  상품 1개 → "single_product" 템플릿
    ▼
[3] AI 처리
    │  Gemini: 캠페인 카피 생성 → "봄바람처럼 가볍게"
    │  Nano Banana Pro: 봄 테마 배경 생성
    ▼
[4] 템플릿 렌더링
    │  슬롯에 값 채우기:
    │  - {{background}} ← AI 생성 배경
    │  - {{campaign_copy}} ← "봄바람처럼 가볍게"
    │  - {{product_image}} ← hat.png
    │  - {{product_name}} ← "클래식 볼캡 스카이블루"
    │  - {{price}} ← "45,000원"
    ▼
[5] Output
    │  편집 가능한 상세페이지
```
